    Critérios de priorização

    Nossos critérios vão de 1 a 5 e seu valor "total" é calculado pela média dos outros 3 critérios (valor de negócio , viabilidade e criticidade)

    Critério Definição Pontuação Significado do Nível
    Valor de Negócio Quanto esse requisito é importante para nossa cliente 1 Contribuição Mínima; este requisito tem um impacto ou benefício insignificante.
    2 Impacto limitado; contribuição moderada aos objetivos de negócio.
    3 Impacto significativo; contribuição substancial aos objetivos de negócio.
    4 Alto impacto; contribuição notável aos objetivos de negócio.
    5 Impacto máximo; contribuição excepcional aos objetivos de negócio.
    Viabilidade Quanto esse requisito não depende de outros + o conhecimento que nossa equipe tem para realizá-lo 1 Pouco viável; recursos e conhecimentos significativamente insuficientes.
    2 Viabilidade limitada; alguns recursos e conhecimentos disponíveis.
    3 Viável; recursos e conhecimentos suficientes para a execução.
    4 Muito viável; recursos e conhecimentos mais do que adequados.
    5 Altamente viável; todos os recursos e conhecimentos necessários estão prontamente disponíveis.
    Criticidade Analisa a urgência deste requisito para atender às necessidades imediatas da cliente. 1 Baixa criticidade; atrasos ou falhas têm pouco impacto no resultado geral.
    2 Criticidade moderada; atrasos ou falhas podem ter algum impacto, mas são gerenciáveis.
    3 Criticidade significativa; atrasos ou falhas têm impacto considerável, mas são recuperáveis.
    4 Muito crítico; atrasos ou falhas têm um impacto substancial e são difíceis de recuperar.
    5 Extremamente crítico; atrasos ou falhas têm um impacto catastrófico e são quase irreparáveis.

    Critérios de Aceitação (MVP 1)

    User Stories Critérios de Aceitação
    [US01] Eu, como administrador, quero ser capaz de realizar o cadastro de produtos no estoque para assim poder realizar as vendas. 1. O administrador deve ser capaz de cadastrar produtos no estoque, informando os seguintes dados: nome do produto, descrição do produto, quantidade em estoque, preço de venda;
    2. O administrador deve ser capaz de visualizar todos os produtos cadastrados no estoque.
    [02] Eu, como administrador, quero ser capaz de editar os produtos no estoque para editar suas informações. 1. O administrador deve ser capaz de editar os dados dos produtos cadastrados no estoque, incluindo: nome do produto, descrição do produto, quantidade em estoque, preço de venda.
    2. Após a edição, a quantidade do produto em estoque deve ser atualizada automaticamente para refletir as mudanças feitas pelo administrador.
    [03] Eu, como administrador, quero ser capaz de excluir produtos no estoque para que assim sejam retirados do banco de dados. 1. O sistema deve permitir que o administrador selecione um produto específico a ser excluído a partir de uma lista completa de produtos no estoque.
    2. Antes de excluir um produto, o sistema deve solicitar uma confirmação do administrador para evitar exclusões acidentais.
    3. O produto deve ser excluído do banco de dados.
    4. O produto deve ser removido da lista de produtos cadastrados.
    [05] Eu, como administrador, quero ser capaz de realizar o cadastro de cada cliente, garantindo que o produto vendido seja vendido para cada cliente específico. 1. O administrador deve ser capaz de cadastrar um cliente.
    2. Durante o cadastro, o sistema deve solicitar informações essenciais do cliente, como nome completo, endereço, número de telefone e e-mail.
    3. O sistema deve garantir que cada cliente cadastrado tenha uma identificação única para evitar duplicatas e garantir integridade dos registros.
    4. O produto vendido deve ser relacionado ao cliente no banco de dados.
    [06] Eu, como administrador, quero ser capaz de realizar a edição de cada cliente para atualizar suas informações. 1. O administrador deve ser capaz de editar os dados dos clientes cadastrados, incluindo: nome, endereço, telefone, e-mail e data de nascimento.
    2. Após a edição, os dados do cliente devem ser atualizados automaticamente no banco de dados e na lista de clientes.
    [07] Eu, como administrador, quero ser capaz de excluir clientes para que assim sejam retirados do banco de dados. 1. O administrador deve ser capaz de excluir clientes do banco de dados.
    2. O cliente deve ser excluído do banco de dados.
    3. O cliente deve ser removido da lista de clientes cadastrados.
    [08] Eu, como administrador, quero ser capaz de realizar o cadastro de cada venda, garantindo que o produto vendido seja retirado do estoque. 1. O administrador deve ser capaz de cadastrar uma venda no sistema.
    2. A venda deve ser registrada no banco de dados.
    3. A quantidade do produto vendido deve ser atualizada no estoque.
    [09] Eu, como administrador, quero ser capaz de realizar a edição de cada venda realizada para assim atualizar suas informações. 1. O administrador deve ser capaz de editar itens de uma venda, como nome do cliente comprador, produto comprado, valor do produto e prazo de entrega.
    2. As informações da venda devem ser atualizadas no banco de dados.
    3. A quantidade do produto vendido deve ser atualizada no estoque.
    [10] Eu, como administrador, quero ser capaz de realizar a exclusão de cada venda realizada para assim ser excluída no banco de dados. 1. O administrador deve conseguir acessar a opção de exclusão de vendas.
    2. A exclusão de uma venda deve resultar na remoção completa dos dados correspondentes no banco de dados.
    3. A venda excluída não deve mais aparecer na lista de vendas após a exclusão.
    [12] Eu, como administrador, quero ser capaz de visualizar um histórico de vendas em ordem cronológica para assim ter um controle melhor das transações/vendas. 1. O administrador deve ter acesso à visualização do histórico de vendas em formato de lista.
    2. As vendas devem ser apresentadas em ordem cronológica, com as mais recentes no topo.
    [18] Eu, como administrador, quero ser capaz de realizar o cadastro de cada despesa, garantindo que o produto comprado seja registrado. 1. O administrador deve ser capaz de cadastrar uma despesa, contendo o valor gasto para confeccionar um determinado artesanato.
    2. A despesa deve ser registrada no banco de dados.
    3. O produto comprado deve ser registrado no estoque.
    [19] Eu, como administrador, quero ser capaz de realizar a exclusão de cada despesa realizada para assim excluí-la no banco de dados. 1. O administrador deve ser capaz de excluir uma despesa.
    2. A despesa deve ser excluída do banco de dados.
    3. A quantidade do produto comprado deve ser atualizada no estoque.
    [20] Eu, como administrador, quero ser capaz de realizar a edição de cada despesa realizada para assim mudar suas informações. 1. O sistema deve permitir que o administrador selecione uma despesa específica a ser editada a partir de uma lista completa de despesas registradas.
    2. O administrador deve ser capaz de editar informações específicas de uma despesa, como descrição, valor, data e categoria.
    3. Após a edição, o balanço financeiro ou relatório de despesas do período deve ser automaticamente atualizado para refletir as mudanças feitas pelo administrador.

    Critérios de Aceitação (MVP 2)

    User Stories Critérios de Aceitação
    [13] Eu, como administrador, quero acessar a plataforma com meu e-mail e senha para garantir a privacidade dos meus dados. 1. O administrador deve conseguir autenticar-se na plataforma utilizando um endereço de e-mail válido e a senha correta.
    2. O sistema não deve realizar o acesso se a senha estiver errada.
    3. O sistema não deve realizar o acesso com campos vazios.
    [14] Eu, como administrador, quero poder fazer logout da minha plataforma de forma segura para assim ter privacidade dos meus dados. 1. O administrador deve ser capaz de fazer logout da plataforma de forma segura.
    2. O logout deve destruir todas as sessões ativas do administrador.
    3. O logout deve impedir que o administrador acesse a plataforma sem inserir novamente seu login e senha.
    [15] Eu, como administrador, quero poder ver minha receita com as vendas para assim poder ter noção de quanto ganhei em determinado mês. 1. O administrador deve ser capaz de visualizar a receita com as vendas de um determinado mês.
    2. A receita deve ser calculada com base no preço de venda dos produtos e na quantidade vendida.
    3. O sistema deve permitir que o administrador selecione um mês e ano específico, para visualizar a receita das vendas durante esse intervalo de tempo.
    [16] Eu, como administrador, quero poder ver meu prejuízo com as vendas para assim poder ter noção de quanto gastei em determinado mês. 1. O administrador deve ser capaz de visualizar o prejuízo com as vendas de um determinado mês.
    2. O prejuízo deve ser calculado com base no preço de compra dos produtos e na quantidade vendida.
    [17] Eu, como administrador, quero poder ver meu lucro para poder ver quanto foi meu ganho real no mês. 1. O administrador deve ser capaz de visualizar o lucro de um determinado mês.
    2. O lucro deve ser calculado com base no preço de venda dos produtos e na quantidade vendida, subtraído do preço de compra dos produtos.
    [04] Eu, como administrador, quero ser alertado quando um material estiver em falta no estoque para assim me atentar em fazer mais desse produto. 1. O administrador deve ser alertado quando a quantidade em estoque de um material cair abaixo de um determinado limite.
    2. O alerta deve ser enviado por e-mail.
    3. O alerta deve conter as seguintes informações: nome do material, quantidade em falta e quantidade mínima.
    [11] Eu, como administrador, quero ser notificado quando o prazo de entrega de uma venda está para expirar para que eu possa tomar as medidas necessárias para garantir que a entrega seja realizada dentro do prazo. 1. O administrador deve ser alertado quando o prazo de entrega de uma venda estiver para expirar.
    2. O alerta deve ser enviado ao administrador por meio de um e-mail.
    3. O alerta deve conter as seguintes informações: número do pedido, data de entrega, produtos encomendados.
